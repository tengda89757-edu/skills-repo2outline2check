# 跨学科研究标准与指南

## 核心原则

### 1. 双重方法论标准

**定义**: AI+实证研究需要同时满足AI技术标准和实证研究规范。

**要求**:
- AI方法需有形式化定义和代码实现
- 实证研究需遵循领域研究规范
- 两方面需有机整合而非简单叠加

**实践**:
```
✅ 正确做法:
- AI方法有理论基础和实现
- 实证设计有方法论依据
- 两方面相互支撑

❌ 错误做法:
- AI部分和实证部分脱节
- 忽视任一方面的标准
- 两张皮式的研究
```

---

### 2. 伦理优先原则

**定义**: 涉及人类被试的研究必须将伦理置于首位。

**要求**:
- 获得IRB/伦理委员会批准
- 实施知情同意
- 保护参与者隐私
- 考虑潜在风险

**实践**:
```
伦理审查要点:
□ 是否需要IRB批准？
□ 知情同意如何实施？
□ 数据如何保护？
□ 弱势群体如何保护？
□ 潜在风险如何缓解？
```

---

### 3. 可复现性标准

**定义**: 研究结果应可被独立研究者复现。

**要求**:
- 公开代码和数据
- 详细的方法描述
- 完整的配置信息
- 固定随机种子

**实践**:
```
可复现性检查清单:
□ 代码是否公开？
□ 数据是否可获取？
□ 环境配置是否完整？
□ 随机种子是否固定？
□ 实验步骤是否详细？
```

---

## 研究设计标准

### RCT设计标准

**适用场景**: 评估AI系统/干预的效果

**必需要素**:
```
1. 随机化
   - 随机化方法说明
   - 分配隐藏实施
   - 基线平衡评估

2. 对照组
   - 对照类型合理
   - 条件对等
   - 盲法（如可行）

3. 样本量
   - 功效分析
   - 最小检测效应
   - 考虑脱落

4. 结局测量
   - 主要结局预指定
   - 测量时点明确
   - 测量者盲法

5. 分析
   - 意向性分析
   - 预指定分析计划
   - 多重比较校正
```

**报告要求**:
- 遵循CONSORT指南
- 提供流程图
- 报告效应量和置信区间

---

### 观察性研究标准

**适用场景**: 无法随机化的情况

**必需要素**:
```
1. 设计选择
   - 队列研究/病例对照/横断面
   - 选择理由说明

2. 暴露测量
   - 暴露定义清晰
   - 测量方法可靠
   - 测量时点明确

3. 结局测量
   - 结局定义清晰
   - 盲法评估（如可行）
   - 随访完整

4. 混杂控制
   - 识别潜在混杂
   - 收集混杂数据
   - 统计调整方法

5. 敏感性分析
   - 验证假设
   - 替代分析方法
```

**报告要求**:
- 遵循STROBE指南
- 讨论残余混杂
- 谨慎因果推断

---

### 混合方法研究标准

**适用场景**: 需要定量和定性数据

**必需要素**:
```
1. 设计类型
   - 聚合式/解释性/探索性
   - 整合策略说明
   - 优先级明确

2. 定量部分
   - 遵循定量研究规范
   - 样本量充足
   - 统计方法恰当

3. 定性部分
   - 遵循定性研究规范
   - 抽样方法说明
   - 编码过程透明

4. 整合分析
   - 整合方法说明
   - 三角验证
   - 一致性/差异讨论
```

**报告要求**:
- 遵循GRMM指南
- 清晰展示整合过程
- 讨论方法论挑战

## 统计报告标准

### 效应量报告

**必需报告**:
```
均值差异:
- Cohen's d (标准化)
- 原始差异 + 置信区间

比例差异:
- Risk Ratio (RR)
- Odds Ratio (OR)
- Risk Difference (RD)

相关:
- Pearson's r
- R² (解释方差)

多变量:
- 偏效应量
- 模型整体效应
```

**解释标准**:
```
Cohen's d:
- 小效应: d = 0.2
- 中效应: d = 0.5
- 大效应: d = 0.8

Pearson's r:
- 小效应: r = 0.1
- 中效应: r = 0.3
- 大效应: r = 0.5
```

---

### 置信区间报告

**格式要求**:
```
均值: M = 45.2, 95% CI [42.1, 48.3]
差异: Δ = 5.3, 95% CI [2.1, 8.5]
效应量: d = 0.65, 95% CI [0.32, 0.98]
```

**解释要点**:
- 说明置信区间含义
- 讨论临床/实际意义
- 与假设检验结果一致

---

### p值报告

**格式要求**:
```
✅ 正确: p = .032, p < .001
❌ 错误: p = .000, p > .05 (ns)

报告精确值（如 p = .032）优于阈值报告
```

**解释要点**:
- p值不等于效应大小
- 不等于结果重要性
- 需结合效应量和置信区间

## AI性能评估标准

### 分类任务

**必需指标**:
```
- Accuracy
- Precision, Recall, F1
- AUC-ROC
- 混淆矩阵

不平衡数据:
- Balanced Accuracy
- F1 (macro/micro)
- AUC-PR
```

---

### 回归任务

**必需指标**:
```
- MAE
- RMSE
- R²
- MAPE (如适用)
```

---

### 生成任务

**必需指标**:
```
文本生成:
- BLEU, ROUGE
- 人工评估
- 语义相似度

图像生成:
- FID, IS
- 人工评估
- 任务特定指标
```

---

### 与人类对比

**要求**:
```
- 报告人类基线
- 说明人类评估者背景
- 报告评估者间一致性
- 讨论AI与人类差异
```

## 伦理审查标准

### IRB审批要求

**需要IRB审批的情况**:
```
□ 涉及人类被试
□ 收集人类数据
□ 分析可识别个人数据
□ 干预人类行为
□ 涉及弱势群体
```

**IRB申请材料**:
```
□ 研究方案
□ 知情同意书
□ 数据收集工具
□ 招募材料
□ 风险评估
□ 数据管理计划
```

---

### 知情同意要求

**必需要素**:
```
□ 研究目的说明
□ 参与内容说明
□ 风险和收益说明
□ 自愿参与声明
□ 退出权利说明
□ 保密承诺
□ 联系方式
□ 特殊人群额外说明
```

---

### 数据保护要求

**技术措施**:
```
□ 数据加密
□ 访问控制
□ 审计日志
□ 安全存储
□ 安全传输
```

**管理措施**:
```
□ 数据管理计划
□ 保留期限规定
□ 销毁程序
□ 共享协议
```

---

## 跨学科整合标准

### 理论整合要求

**必需要素**:
```
1. 两领域理论基础
   - AI理论引用
   - 领域理论引用
   - 理论解释准确

2. 整合框架
   - 框架图清晰
   - 映射关系明确
   - 逻辑合理

3. 理论驱动假设
   - 假设有理论依据
   - 假设可检验
   - 与实验对应
```

---

### 文献综述要求

**平衡原则**:
```
- AI文献和领域文献比例合理
- 引用两领域经典文献
- 引用最新跨学科研究
- 识别跨学科研究空白
```

---

### 贡献声明要求

**适度原则**:
```
✅ 正确做法:
- 明确技术贡献
- 明确领域贡献
- 说明跨学科价值
- 不过度声明

❌ 错误做法:
- 夸大任一方面贡献
- 忽视另一方面贡献
- 声明无法验证
```
