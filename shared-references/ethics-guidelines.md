# AI+实证研究伦理指南

## 核心伦理原则

### 1. 尊重人格

**原则**: 尊重研究参与者的自主权和尊严。

**要求**:
- 获得知情同意
- 保护隐私和保密
- 尊重退出权利
- 保护弱势群体

**实践**:
```
知情同意检查清单:
□ 使用参与者可理解的语言
□ 说明研究目的和程序
□ 说明潜在风险和收益
□ 说明数据使用和保护
□ 说明参与是自愿的
□ 说明可随时退出
□ 提供联系方式
```

---

### 2. 行善与不伤害

**原则**: 最大化利益，最小化伤害。

**要求**:
- 评估潜在风险
- 设计风险缓解措施
- 确保收益大于风险
- 持续监控安全

**实践**:
```
风险评估框架:
1. 识别潜在风险
   - 身体风险
   - 心理风险
   - 社会风险
   - 经济风险
   - 隐私风险

2. 评估风险概率和严重程度

3. 设计缓解措施

4. 评估剩余风险是否可接受
```

---

### 3. 公正

**原则**: 公平分配研究收益和负担。

**要求**:
- 公平选择参与者
- 公平分配研究收益
- 避免剥削弱势群体
- 考虑研究结果的公平影响

**实践**:
```
公正性检查:
□ 参与者选择是否公平？
□ 研究收益是否公平分配？
□ 是否有群体被不当排除？
□ 研究结果是否可能加剧不平等？
```

## 领域专属伦理考量

### AI+教育伦理

#### 学生保护

```
□ 未成年人保护
  - 获得家长/监护人同意
  - 使用适龄语言解释
  - 避免过度收集数据
  - 限制数据使用范围

□ 学习环境影响
  - 不干扰正常教学
  - 不增加学生负担
  - 提供等效替代方案
  - 考虑长期影响

□ 教育公平
  - 考虑数字鸿沟
  - 避免算法偏见
  - 确保可及性
  - 关注弱势学生
```

#### 数据保护

```
□ 教育数据特殊性
  - 学习数据敏感
  - 成绩数据敏感
  - 行为数据敏感
  - 长期影响考虑

□ FERPA合规（美国）
  - 教育记录保护
  - 家长访问权
  - 学生同意年龄
  - 数据披露限制

□ GDPR合规（欧盟）
  - 儿童数据特殊保护
  - 数据最小化原则
  - 被遗忘权
  - 数据可携带权
```

#### 教育伦理声明模板

```markdown
## Ethics Statement

### Participant Protection
This study involved [student age range] students. We obtained informed 
consent from parents/guardians and assent from students. The study 
was approved by [Institution] IRB (Protocol #XXXX).

### Data Protection
All student data were de-identified and stored securely. Only 
aggregated results are reported. Data will be retained for [X] years 
and then destroyed.

### Educational Equity
We considered potential impacts on educational equity. The AI system 
was designed to be accessible to students with diverse backgrounds 
and abilities. We report results disaggregated by demographic groups 
to identify potential disparities.

### Teacher Involvement
Teachers were involved in the design and implementation of the study. 
Their feedback was incorporated to ensure the intervention was 
educationally appropriate.
```

### AI+医学伦理

#### 患者保护

```
□ 临床试验伦理
  - 遵循赫尔辛基宣言
  - 独立伦理委员会审查
  - 安全监控
  - 数据安全监测委员会

□ 患者安全
  - 风险最小化
  - 不延误标准治疗
  - 不良事件报告
  - 紧急情况预案

□ 弱势患者
  - 严重疾病患者
  - 认知障碍患者
  - 儿童患者
  - 经济弱势患者
```

#### 数据保护

```
□ HIPAA合规（美国）
  - PHI保护
  - 最小必要原则
  - 患者权利
  - 业务伙伴协议

□ 医疗数据特殊性
  - 高度敏感
  - 终身影响
  - 重新识别风险
  - 家族关联

□ 临床数据共享
  - 去标识化标准
  - 安全数据环境
  - 访问控制
  - 使用审计
```

#### 医学伦理声明模板

```markdown
## Ethics Statement

### Ethical Approval
This study was approved by [Institution] IRB (Protocol #XXXX) and 
conducted in accordance with the Declaration of Helsinki. The study 
was registered at ClinicalTrials.gov (NCTXXXXXX).

### Informed Consent
All participants provided written informed consent. For patients 
with impaired decision-making capacity, consent was obtained from 
legally authorized representatives.

### Safety Monitoring
An independent Data Safety Monitoring Board reviewed safety data 
at [frequency]. No serious adverse events were attributed to the 
AI system.

### Clinical Implications
The AI system is intended as a clinical decision support tool and 
does not replace clinical judgment. We discuss the potential risks 
of over-reliance on AI recommendations in the manuscript.

### Data Protection
All patient data were de-identified according to HIPAA Safe Harbor 
guidelines. Data were stored in a secure, access-controlled environment.
```

## 伦理审查流程

### 提交前自查

```
Step 1: 确定是否需要伦理审查
□ 是否涉及人类被试？
□ 是否收集人类数据？
□ 是否分析可识别数据？
□ 是否涉及干预？

Step 2: 准备伦理申请
□ 研究方案
□ 知情同意书
□ 数据收集工具
□ 招募材料
□ 风险评估
□ 数据管理计划

Step 3: 提交审查
□ 选择合适的审查类型
□ 准备完整材料
□ 回应审查意见

Step 4: 研究实施
□ 获得批准后开始
□ 遵循批准方案
□ 报告方案偏离
□ 持续监控风险
```

### 高风险研究处理

**高风险判定标准**:
```
□ 涉及弱势群体
□ 涉及敏感话题
□ 可能造成重大伤害
□ 可能影响社会公平
□ 可能被滥用
```

**高风险研究处理**:
```
1. 完整委员会审查（非简易审查）
2. 额外保护措施
3. 持续监控
4. 定期报告
5. 可能需要外部咨询
```

## 常见伦理问题

### Q1: 使用公开数据是否需要IRB？

```
答: 取决于数据性质

需要IRB:
- 数据包含可识别信息
- 数据使用超出原始同意范围
- 数据涉及敏感信息

可能不需要IRB:
- 完全去标识化的公开数据
- 数据使用符合原始同意
- 不涉及人类被试干预

建议: 咨询IRB确定
```

### Q2: AI系统评估是否需要IRB？

```
答: 取决于评估方式

需要IRB:
- 涉及真实用户
- 收集用户数据
- 影响用户决策

可能不需要IRB:
- 仅使用历史数据
- 无用户参与
- 纯技术评估

建议: 咨询IRB确定
```

### Q3: 如何处理意外发现？

```
答: 预先规划

1. 在方案中说明可能发现的类型
2. 制定处理流程
3. 考虑报告义务
4. 考虑参与者福利
5. 必要时修改方案
```
