## 不同期刊格式要求

### AI顶会格式 (NeurIPS/ICLR/ICML)

```
- 页数限制: 通常8-9页正文 + 无限附录
- 匿名投稿: 是
- 审稿标准: 技术创新性、实验严谨性
- 特殊要求: 
  - 强调算法创新
  - 需要与SOTA对比
  - 开源代码期望
```

### 教育期刊格式 (Computers & Education, BJET)

```
- 字数限制: 通常6000-8000词
- 结构要求: IMRaD结构
- 审稿标准: 教育价值、方法严谨性
- 特殊要求:
  - 强调学习效果
  - 需要教育理论支撑
  - 讨论教育启示
```

### 医学期刊格式 (JAMIA, NPJ Digital Medicine)

```
- 字数限制: 通常3000-5000词
- 结构要求: 结构化摘要，IMRaD
- 审稿标准: 临床价值、方法严谨性
- 特殊要求:
  - 遵循报告指南(CONSORT/STROBE)
  - IRB批准必须
  - 讨论临床意义
```

### 社科期刊格式 (JCSS, Computational Sociology)

```
- 字数限制: 通常8000-12000词
- 结构要求: 灵活，但需逻辑清晰
- 审稿标准: 理论贡献、方法严谨性
- 特殊要求:
  - 强调理论贡献
  - 方法论反思
  - 伦理讨论充分
```

## 写作风格指南

### 跨学科写作原则

```
1. 术语平衡
   - 技术术语需解释
   - 领域术语需定义
   - 避免任一领域的行话

2. 引用平衡
   - AI文献和领域文献比例合理
   - 引用两领域经典文献
   - 引用最新跨学科研究

3. 贡献声明平衡
   - 明确技术贡献
   - 明确领域贡献
   - 强调跨学科价值

4. 讨论平衡
   - 技术视角解释
   - 领域视角解释
   - 跨学科整合解释
```

### 常见问题避免

```
❌ 过于技术化
   - 领域读者无法理解
   - 忽视领域问题复杂性

❌ 过于应用化
   - 技术读者觉得浅显
   - 缺乏技术创新

❌ 两张皮
   - AI部分和领域部分脱节
   - 缺乏真正的整合

❌ 贡献不清
   - 技术贡献和领域贡献混淆
   - 跨学科价值不明确

✅ 正确做法
   - 两领域读者都能理解核心内容
   - AI技术和领域问题有机结合
   - 贡献清晰，价值明确
```
